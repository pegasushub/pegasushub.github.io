Run & Contribute

The complete lifecycle: Scaffold your projects, Execute on grid resources, and Publish your workflows.

Goal of Deployment

Execute & Contribute

This page provides the necessary tools to take your workflow from a local concept to grid-scale execution. Use modern scaffolding templates to generate your project, choose your execution environment, and finally, share your scientifically verified workflow with the global PegasusHub community.

Step 1: Initialization

Scaffold New Project

Rapidly bootstrap a new Pegasus workflow project using our official generation templates.

Why use Scaffolding?

It eliminates manual setup errors. These tools instantly generate a standardized directory structure, boilerplate Python scripts, and the pegasus.properties file.

Copier vs Cookiecutter

Cookiecutter is the classic one-time generator. Copier is recommended because it supports project lifecycle updates, allowing you to seamlessly pull upstream template improvements into your existing project later.

View Template Repository
Copier Engine Recommended
pip install copier
copier copy https://github.com/swarmourr/pegasus-wms-template.git ./my-project
Cookiecutter Engine Classic
pip install cookiecutter
cookiecutter https://github.com/swarmourr/pegasus-wms-template.git
